¿Qué es Git?

Se trata de un sistema de control de versiones distribuido. Esto significa que cualquier desarrollador del equipo que tenga acceso puede gestionar el código fuente y su historial de cambios utilizando las herramientas de línea de comandos de Git.

A diferencia de los sistemas de control de versiones centralizados, Git ofrece ramas de características. Esto significa que cada ingeniero de software en el equipo puede dividir una rama de características que proporcionará un repositorio local aislado para hacer cambios en el código.

¿Qué es Github?

GitHub es un servicio basado en la nube que aloja un sistema de control de versiones (VCS) llamado Git. Éste permite a los desarrolladores colaborar y realizar cambios en proyectos compartidos, a la vez que mantienen un seguimiento detallado de su progreso.

Flujo de comandos


Comando

Descripción
Git clone Sirve para clonar un repositorio de Github, es decir, descarga todo el repositorio y su historial de versiones.
Git fetch -a Sirve para descargar todos los cambios que han habido en el repositorio remoto, pero no se aplica al código.
Git pull Sirve para descargar y fusionar los cambios remotos con tu código local.
Git checkout -b "nombre de la rama" Sirve para crear una nueva rama y cambia automáticamente a esta.
Git add Sirve para agregar los archivos que hayas modificado a tu branch
  • -A: añade todos los cambios que se hayan hecho, en cualquier parte del repositorio.
  • . : añade todos los cambios que hayas hecho, en tu directorio actual.
  • "nombre del archivo": añade todos los cambios que hayas hecho en un archivo en especifico.
Git commit -m "mensaje" Sirve para guardar los cambios en tu repositorio local, el mensaje tiene que tener una breve descripción de tus cambios.
Git push -u origin "nombre de la rama" Sirve para subir los cambios que hiciste en una rama de tu repositorio local al repositorio remoto.
Git pull origin main Después de cambiar de rama, sirve para asegurarnos que tienes todos los cambios en la rama.
Git merge "nombre de la rama" Sirve para unir el contenido de otra rama a la rama en la que tu estas actualmente.

Otros comandos útiles

  • Git status: Sirve para mostrar el estado actual del repositorio. Indica cuales archivos han cambiado, cuales están en staging, etc.
  • Git log: Sirve para mostrar el historial del repositorio.
  • Git restore: Sirve para deshacer cambios en los archivos antes de hacer un commit.
  • Git switch "nombre de la rama": Sirve para cambiar entre ramas.
