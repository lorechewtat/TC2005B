  1. ¿Qué es el DOM?

    2. Respuesta Copilot

    El DOM (Document Object Model) es una interfaz de programación para documentos HTML y XML. Representa la estructura del documento como un árbol de nodos, donde cada nodo corresponde a una parte del documento, como un elemento, atributo o texto. Esto permite a los lenguajes de programación, como JavaScript, manipular dinámicamente el contenido, la estructura y el estilo de la página web.

    3. Respuesta Copilot

  3. ¿Cómo se accede a un elemento del DOM?

    4. Respuesta Copilot

    Para acceder a un elemento del DOM, se pueden utilizar varios métodos en JavaScript. Uno de los más comunes es getElementById, que permite seleccionar un elemento específico utilizando su atributo id. También se puede usar getElementsByClassName para obtener una colección de elementos que comparten una misma clase, o getElementsByTagName para seleccionar todos los elementos de un tipo de etiqueta específico, como <div> o <p>. Además, los métodos querySelector y querySelectorAll permiten seleccionar elementos utilizando selectores CSS; el primero devuelve el primer elemento que coincide con el selector, mientras que el segundo devuelve todos los elementos coincidentes. Estos métodos son esenciales para manipular dinámicamente el contenido, los estilos y los atributos de los elementos en una página web.

    Respuesta ChatGPT

Se puede seleccionar un elemento por su ID con getElementById, obtener una colección de elementos con una misma clase usando getElementsByClassName, seleccionar elementos por su etiqueta con getElementsByTagName, o emplear selectores CSS con querySelector para obtener el primer elemento coincidente o querySelectorAll para obtener todos los elementos que coincidan con el selector.

  4. ¿Cómo se modifica un elemento del DOM?

    5. Respuesta Copilot

    Para modificar un elemento del DOM, primero necesitas seleccionarlo utilizando métodos como getElementById, querySelector o similares. Una vez seleccionado, puedes cambiar sus propiedades o contenido utilizando propiedades como innerHTML, textContent, o estilos con style. Por ejemplo, para cambiar el texto de un elemento con un ID específico, puedes usar: document.getElementById('miElemento').textContent = 'Nuevo texto';.

  5. ¿Cuál es la diferencia entre getElementby* y querySelector*?

    6. Respuesta Copilot

    La diferencia entre getElementBy* y querySelector* radica en cómo seleccionan elementos del DOM. Los métodos getElementById, getElementsByClassName y getElementsByTagName son más específicos y rápidos, pero menos flexibles, ya que solo permiten seleccionar elementos por ID, clase o etiqueta, respectivamente. Por otro lado, querySelector y querySelectorAll utilizan selectores CSS, lo que los hace más versátiles, ya que puedes combinar clases, IDs, pseudoclases, y más, aunque suelen ser un poco más lentos.

    Respuesta ChatGPT

    La diferencia entre los métodos getElementBy y querySelector radica en que los primeros son métodos más antiguos y específicos, diseñados para seleccionar elementos por ID, clase o etiqueta, mientras que querySelector y querySelectorAll permiten el uso de selectores CSS y ofrecen mayor flexibilidad al permitir seleccionar el primer elemento coincidente o todos los elementos que cumplan con el selector.

  6. ¿Qué es un evento?

    7. Respuesta Copilot

  7. ¿Qué es un listener?

    8. Respuesta Copilot

    Un listener (o event listener) es una función que "escucha" un evento específico en un elemento del DOM. Cuando ese evento ocurre, el listener ejecuta una función asociada, permitiendo que el código responda a las acciones del usuario.

    Respuesta ChatGPT

    Para responder a eventos, se emplean los listeners, que son funciones que se encargan de "escuchar" un evento y ejecutar un código cuando este ocurre. Estos listeners se pueden agregar a un elemento mediante el método addEventListener, especificando el tipo de evento y la función a ejecutar.

  8. ¿Qué es un callback?

    9. Respuesta Copilot

    Un callback es una función que se pasa como argumento a otra función y que se ejecuta después de que se complete una acción o evento. En el contexto de eventos, el callback es la función que se ejecuta cuando el listener detecta el evento.

    Respuesta ChatGPT

    Un callback es una función que se pasa como argumento a otra función para ser ejecutada posteriormente. Un ejemplo común de callback ocurre con la función setTimeout, donde la función que se pasa como argumento se ejecutará después de un tiempo determinado.

  9. ¿Qué es un objeto de evento?

    10. Respuesta Copilot

    Un objeto de evento es un objeto que contiene información sobre un evento que ha ocurrido. Este objeto se pasa automáticamente al callback asociado al evento y proporciona detalles como el tipo de evento, el elemento que lo generó, las coordenadas del mouse (si aplica), y más, permitiendo manejar el evento de manera más precisa.

    Respuesta ChatGPT

    Un objeto de evento es un objeto que se genera automáticamente cuando ocurre un evento y contiene información sobre él, como el tipo de evento, el elemento que lo desencadenó y detalles adicionales. En un evento de clic, por ejemplo, el objeto de evento permite identificar qué elemento fue clickeado a través de la propiedad target. Este objeto se puede recibir como parámetro en la función del listener para acceder a su información y responder de manera adecuada al evento.

Conclusiones

Experiencia general

En general, creo que fue un buen ejercicio para ver cuáles realmente son las diferencias entre las diferentes inteligencias artificiales. Además, creo que es un buen ejercicio para también ver como hacen código las IAs y las diferencias que podrían tener con el código de un humano. Además, también siento que fue un buen ejercicio debido a que en algunas cosas siento que me hubiera tardado menos en hacerlas yo que en pedirle a la IA que lo hiciera. Igualmente, creo que me ayudó a ver que las IAs no son tan precisas con sus respuestas y que a veces no son tan útiles como uno pensaría. En general, creo que es un buen ejercicio para ver como funcionan las IAs y en que si te servíria usarlas en el desarrollo de software.

¿Cuándo y para qué usarías ChatGPT y cuándo Copilot?

Según mi experiencia en esta actividad y en otras donde he usado la ayuda de alguna IA, creo que ChatGPT es mucho mejor para corregir errores en un código que ya tienes, o para explicarte como funciona algo. Esto debido a que varias veces cuando le pedía las definiciones a Copilot pensaba que era implementarlo directo en el código en vez de darme la respuesta de las preguntas. Además, también creo que ChatGPT es una buena herramienta para ayudarte a optimizar el código o darte soluciones para hacer cambios en el código. En cambio, creo que Copilot es mejor para generar código desde cero, ya que es más rápido y tiene acceso directo a tu código y los cambios que le hagas en todo momento, a diferencia de ChatGPT que solo tiene acceso al código si se lo pasas, pero si le haces algún cambio, no lo va a saber a menos de que se lo digas. Además, creo que Copilot es una buena herramienta para optimizar procesos tediosos como escribir código repetitivo o cambiar varios elementos de un código. Por ejemplo, en esta actividad, Copilot me ayudó a poner todos los h3 en los elementos li, y eso me ahorró mucho tiempo.

Ventajas de usar AI para programar

Teniendo en cuenta lo que comente en el punto anterior, las ventajas de usar IA para programar son varias. Algunas de las más importantes son:

  • Te ayuda a optimizar procesos tediosos como cambiar elementos que se repiten varias veces.
  • Te ayuda a encontrar errores en el código que tal vez te hubieras tardado mucho en encontrar.
  • Te ayuda a entender mejor como funciona el código, debido a que si no entiendes puedes preguntarle qué hace cada parte del código.
  • Te ayuda a generar código desde cero.
  • Te ayuda a optimizar el código cuando ya esta terminado.

Desventajas de usar AI para programar

Teniendo en cuenta lo que comente en el punto anterior, las desventajas de usar IA para programar son también son varias. Algunas de las más importantes son:

  • Puede generar código que no tiene sentido con lo que quieres hacer. Es decir, varias veces la IA no entiende muy bien que quieres hacer, y puedes tardarte horas intentandole explicar que quieres que haga.
  • Puede generar código que no es correcto. Por ejemplo, Copilot no pudo resolver el último ejercicio a la primera, entonces le tuve que pedir ayuda a ChatGPT para que me ayudara a resolverlo. Igualmente, no supo cuál era el error y me tardé mucho tiempo intentando hacer que funcionara, hasta que decidí checar el código yo para darme cuenta que era un problema con el link.
  • Puede generar código que no es compatible con otras partes del código. Es decir, si tienes más código y no le pasas el contexto completo, la IA puede hacer algo completamente distinto con lo que ya tienes.
  • Te quita creatividad. Al querer optimizar todo, muchas veces perdemos la originalidad que tendría el código de un humano. Este creo que es la desventaja mas importante, ya que al final del día, si solo usaramos IA para programar, no sería necesario el uso de un humano para programar, y esto también podría llevar a que todos los códigos fueran iguales. Si fuera asi, no veríamos las diferentes formas en las que los humanos pueden pensar para resolver un problema. Creo que es importante que solo se use la IA como una herramienta, en vez de usarla como una solución a todos los problemas.
